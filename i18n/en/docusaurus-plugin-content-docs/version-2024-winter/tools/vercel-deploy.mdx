## Deploying Projects

### Preparation

<div className="prerequisites">

#### Requirements
- Git repository (GitHub, GitLab, or Bitbucket)
- Project package.json file
- Node.js environment

#### Recommended Configuration
```json
{
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start"
  }
}
```

</div>

### Importing Project

<div className="step-guide">

#### 1. Connect Git Platform

<Tabs>
  <TabItem value="github" label="GitHub" default>
    1. Visit [Vercel Dashboard](https://vercel.com/dashboard)
    2. Click "New Project"
    3. Select "Import Git Repository"
    4. Authorize GitHub access
  </TabItem>
  <TabItem value="gitlab" label="GitLab">
    1. Visit Vercel Dashboard
    2. Click "Add GitLab Account"
    3. Complete GitLab authorization
    4. Select repository to import
  </TabItem>
</Tabs>

#### 2. Configure Project

```yaml
# vercel.json configuration example
{
  "version": 2,
  "builds": [
    {
      "src": "package.json",
      "use": "@vercel/next"
    }
  ],
  "routes": [
    { "src": "/(.*)", "dest": "/" }
  ]
}
```

#### 3. Deployment Settings

- Select project framework (usually auto-detected)
- Configure build command and output directory
- Set environment variables (if needed)

</div>

## Advanced Feature Configuration

### Custom Domain

<div className="domain-setup">

#### Adding Domain
1. Find "Domains" in project settings
2. Click "Add Domain"
3. Enter your domain name

#### DNS Configuration
```bash
# A Record configuration
Type: A
Name: @
Value: 76.76.21.21

# CNAME configuration
Type: CNAME
Name: www
Value: cname.vercel-dns.com
```

</div>

### Environment Variable Management

<div className="env-management">

#### Setting in Vercel Dashboard
1. Project Settings > Environment Variables
2. Add key-value pairs
3. Select environment (Production/Preview/Development)

#### Using .env Files
```bash
# .env.local
DATABASE_URL=xxx
API_KEY=xxx

# .env.production
NODE_ENV=production
API_ENDPOINT=https://api.example.com
```

:::caution Note
Do not commit .env files containing sensitive information to Git repository!
:::

</div>

## Best Practices

### Optimizing Build Process

<div className="optimization-tips">

#### Caching Strategy
```json
{
  "build": {
    "env": {
      "NEXT_PUBLIC_BUILD_TIME": "@now"
    }
  },
  "headers": [
    {
      "source": "/static/(.*)",
      "headers": [
        {
          "key": "Cache-Control",
          "value": "public, max-age=31536000, immutable"
        }
      ]
    }
  ]
}
```

#### Build Optimization
- Use dependency caching
- Optimize build commands
- Configure output directory properly
