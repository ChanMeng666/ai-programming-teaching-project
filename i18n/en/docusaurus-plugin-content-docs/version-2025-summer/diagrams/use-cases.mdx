# Diagram Application Guide for Different Scenarios

## üìö Academic Paper Diagram Standards

### Journal Submission Requirements

<Admonition type="important" title="General Academic Standards">
- **Vector Format**: Use SVG or EPS format to ensure print quality
- **Font Standards**: Times New Roman or Arial, minimum 8pt size
- **Numbering Rules**: Figure 1, Figure 2... consecutive numbering
- **Citation Format**: Use "as shown in Figure 1" or "see Figure 1" in the text
- **Resolution Requirements**: Bitmap minimum 300 DPI
- **Color Mode**: CMYK for print, RGB for digital
</Admonition>

<Admonition type="tip" title="Diagram Organization Recommendations">
1. **Layered Display**: Progress from high-level architecture to detailed design layer by layer
2. **Unified Style**: Maintain consistent colors, fonts, and icon styles
3. **Version Control**: Include diagram source files in Git
4. **Synchronized Updates**: Update diagrams promptly when code changes
</Admonition>

## üì± Responsive Design

### Mobile Optimization

<Admonition type="tip" title="Mobile Diagram Design Principles">
- **Simplify Content**: Reduce number of nodes, highlight core information
- **Enlarge Elements**: Ensure touch targets are at least 44√ó44 pixels
- **Vertical Layout**: Prioritize top-to-bottom layout
- **Scalable**: Use SVG format to support zooming
- **Segmented Display**: Split complex diagrams into multiple simple diagrams
</Admonition>
