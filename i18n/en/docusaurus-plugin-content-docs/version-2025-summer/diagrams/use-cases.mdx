# Diagram Application Guide for Different Scenarios

## Academic Paper Diagram Standards

### Journal Submission Requirements

<Admonition type="important" title="General Academic Standards">
- **Vector Format**: Use SVG or EPS format to ensure print quality
- **Font Standards**: Times New Roman or Arial, minimum 8pt size
- **Numbering Rules**: Figure 1, Figure 2... consecutive numbering
- **Citation Format**: Use "as shown in Figure 1" or "see Figure 1" in the text
- **Resolution Requirements**: Bitmap minimum 300 DPI
- **Color Mode**: CMYK for print, RGB for digital
</Admonition>

### IEEE Paper Diagram Standards
#### Size Requirements
    - Single column: 3.5 inches wide
- Double column: 7.16 inches wide
- Height: not exceeding 9.5 inches
#### Label Standards
    - Caption: Below figure, centered
- Font: 8-10pt
- Line width: 0.5-1pt

### Converting Mermaid Diagrams to Academic Format

<CodeBlock language="text">
{`You: Convert my Mermaid flowchart to a format suitable for IEEE papers

Gemini: I'll help you convert it to academic paper format:

1. Export as SVG:
   - Mermaid Live Editor → Actions → Download SVG
   - Save as figure1_flowchart.svg

2. Optimize SVG:
   - Open with Inkscape
   - Set document size to 3.5 inches (single column)
   - Change font to Arial 10pt
   - Adjust line width to 0.75pt
   - Export as EPS format

3. LaTeX code:
\\begin{figure}[t]
  \\centering
  \\includegraphics[width=\\columnwidth]{figure1_flowchart.eps}
  \\caption{Proposed Algorithm Workflow}
  \\label{fig:workflow}
\\end{figure}`}
</CodeBlock>

### API Documentation Diagrams
#### Recommended Diagram Types
##### Sequence Diagrams
Show API call flow
Suitable for: Authentication, Payment flows
##### Flowcharts
Explain business logic
Suitable for: Data processing, Decision flows
##### State Diagrams
Describe resource state changes
Suitable for: Order status, Task status
### System Design Documentation

<Admonition type="tip" title="Diagram Organization Recommendations">
1. **Layered Display**: Progress from high-level architecture to detailed design layer by layer
2. **Unified Style**: Maintain consistent colors, fonts, and icon styles
3. **Version Control**: Include diagram source files in Git
4. **Synchronized Updates**: Update diagrams promptly when code changes
</Admonition>

## PPT Presentation Optimization

### Step-by-Step Display Techniques
#### Progressive Display
    1. Slide 1: Core concepts
2. Slide 2: Add details
3. Slide 3: Complete architecture
4. Slide 4: Emphasize key points
#### Animation Effects
    - Fade in: New components
- Path animation: Data flow
- Color change: State transitions
- Zoom: Focus switching

## Responsive Design

### Mobile Optimization

<Admonition type="tip" title="Mobile Diagram Design Principles">
- **Simplify Content**: Reduce number of nodes, highlight core information
- **Enlarge Elements**: Ensure touch targets are at least 44×44 pixels
- **Vertical Layout**: Prioritize top-to-bottom layout
- **Scalable**: Use SVG format to support zooming
- **Segmented Display**: Split complex diagrams into multiple simple diagrams
</Admonition>
