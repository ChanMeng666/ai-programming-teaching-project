# 面向求职的 Typst PDF 自动化

## 学习目标

到本节课结束，学员应当能够：

- **通过 Claude Code 调用 Typst skill**，把 Neon 里结构化的 profile 数据转成一份像素级精致、可下载的 PDF —— 一行 Typst 都不用自己写。
- **让 Neon `profile` 成为一个数据源，驱动两种产物** —— 作品集站和简历 / 求职信 PDF —— 证明 **数据在前、文档在后** 是一种资深工程师的习惯。
- **在线上作品集上交付两个按钮**：一键 "Download CV"，以及 "Generate cover letter for \[job description\]" —— 按需产生定制 PDF。

## 核心主题

- 一句话说 Typst：像"带超能力的 Markdown，能编译成 PDF"。这堂课的讲义就是用它编出来的。
- Claude Code 里的 Skills —— Typst skill 是一个**开源 skill**，我们在阶段 2 自己装上（Claude Code **不会** 自动发现 skill）。装好之后，它就会教 Claude Code 如何从结构化输入生成常见文档。
- 数据驱动的文档：为什么把 CV 字段存在 Neon（而不是 Google Docs）能让它被 AI 编辑一辈子。
- 求职信是 LLM 的完美任务：结构化输入（你的 profile + JD）+ 结构化输出（格式化的 PDF）。
- Meta 时刻：学员意识到这节讲义和他们的简历是同一种工具做出来的。

## 工具 / 技术栈

| 工具 | 本周角色 |
|---|---|
| **Typst CLI** | 把 `.typ` 文件编译成 PDF —— 快且确定。 |
| **Typst skill**（Claude Code） | AI 写 `.typ` 模板并驱动编译。 |
| **Neon Postgres + Drizzle** | 你 `profile` 行的存放地：一个数据源。 |
| **Claude API**（可选） | 根据 JD 生成定制求职信文本。 |
| **Vercel serverless function** | 按需跑 Typst 编译并流式返回 PDF。 |

## 动手 Lab

**任务。** 下课时你的作品集多出两个按钮：**Download CV**（按 Neon profile 生成一份打磨过的一页 CV）和 **Generate cover letter**（粘一段 JD，拿一份定制 PDF）。两者都通过 Typst skill 编译。

### 阶段 1 —— 设计 profile 数据模型

<PromptStep n={1} audience="Cursor">
在 Neon 里加一张 `profile` 表，它将作为我 CV 数据的唯一真实数据源。帮我设计 Drizzle schema。**每个用户一行 profile**——也就是我会是唯一那个有数据的 row，但未来的多用户版本要支持更多。包含：

- `user_id`（外键 → users.id，unique）—— 和 user 一对一
- `headline`（text，一行，如 "CS undergrad building AI agents"）
- `summary`（text，3–5 句）
- `email`、`phone`（可空）、`location_city`、`location_country`、`website`、`github`、`linkedin`（都是 text，都可空）
- `education`（JSON：`{school, degree, start, end, notes}` 数组）
- `experience`（JSON：`{org, role, start, end, highlights: []}` 数组）
- `projects`（JSON：`{name, url, one_liner, stack: []}` 数组）
- `skills`（JSON：字符串数组）
- `updated_at`（timestamp 自动更新）

生成并 push 迁移。再在站点上加一个 admin-only 的 `/edit-profile` 页，让我能用 textarea 或 JSON 编辑器改每个字段。只有登录后的我能访问这页。
</PromptStep>

<VerifyStep n={2}>

- 迁移已应用；`profile` 表存在。
- 以我登录状态访问 localhost `/edit-profile` 能看到编辑器。
- 未登录 —— 或以别的用户登录 —— 重定向回首页。
- 存一份最小 profile（headline + summary + 一条 education + 一条 experience）。确认 row 在 Neon。

</VerifyStep>

### 阶段 2 —— 安装 Typst CLI 和 Typst skill

<Note title="要装两个东西 —— 它们是不同的">
Typst **CLI** 是把 `.typ` 编译成 PDF 的二进制；Typst **skill** 是一包指令、参考资料和示例，教 Claude Code 怎么 *驱动* 那个 CLI。Claude Code **不会** 从任何"Anthropic registry"里自动发现 skill —— 你要自己装，就像装 VS Code 插件那样。阶段 2 把这两件事都做完。
</Note>

#### 步骤 2a —— 安装 Typst CLI

挑一条匹配你机器的命令跑一次：

```bash
# Windows
winget install --id Typst.Typst

# macOS
brew install typst

# 任何装了 Rust 工具链的平台
cargo install --locked typst-cli
```

跑 `typst --version` 验一下 —— 应该看到 `typst 0.14.x` 或更新。

#### 步骤 2b —— 安装 Typst skill

这个 skill 住在公开 GitHub 仓库：[`ChanMeng666/typst-claude-skill`](https://github.com/ChanMeng666/typst-claude-skill)。下面四种安装方式**任挑一种**——能力上完全等价。

<Tabs>
  <Tab title="1. Plugin marketplace（推荐）">
在 Claude Code 内跑两条 slash 命令：

```text
/plugin marketplace add ChanMeng666/typst-claude-skill
/plugin install typst@chan-typst
```

plugin `version` 字段升级时会自动更新。marketplace 名字（`chan-typst`）和 plugin 名字（`typst`）声明在仓库的 `.claude-plugin/marketplace.json` 与 `.claude-plugin/plugin.json` 里。
  </Tab>
  <Tab title="2. Git clone（适合想 hack skill 的同学）">
直接走 git。仓库里真正的 skill 内容住在 `skills/typst/` 子目录，所以要先把仓库克隆到临时位置，再把那个子目录搬到 skills 文件夹：

```bash
# macOS / Linux / Windows（Git Bash 或 WSL）
git clone --depth 1 https://github.com/ChanMeng666/typst-claude-skill.git /tmp/_typst-src \
  && mv /tmp/_typst-src/skills/typst ~/.claude/skills/typst \
  && rm -rf /tmp/_typst-src
```

> Windows PowerShell：用 `$env:USERPROFILE` 替代 `~`，用 `Move-Item` 替代 `mv`，用 `Remove-Item -Recurse -Force` 替代 `rm -rf`。
  </Tab>
  <Tab title="3. 手动 ZIP">
机器上没有 Git 的同学：

1. 打开 [`github.com/ChanMeng666/typst-claude-skill`](https://github.com/ChanMeng666/typst-claude-skill)，点 **Code → Download ZIP**。
2. 解压。
3. 进到解压出来的文件夹里，找到 `skills/typst/` 子目录 —— 这个子目录**就是** skill。
4. 把 `skills/typst/` 移动并重命名为 `~/.claude/skills/typst`（macOS / Linux）或 `%USERPROFILE%\.claude\skills\typst`（Windows）。
5. 确认 `SKILL.md` 在目标文件夹的根。
  </Tab>
  <Tab title="4. 项目级（per-project）">
当 skill 应该**只**在这个项目内可用时使用（和项目一起 commit，让 clone 的人都自动拿到）：

```bash
# 从项目根目录跑
git clone --depth 1 https://github.com/ChanMeng666/typst-claude-skill.git /tmp/_typst-src \
  && mkdir -p .claude/skills \
  && mv /tmp/_typst-src/skills/typst .claude/skills/typst \
  && rm -rf /tmp/_typst-src
```

skill 只在 Claude Code 从这个项目目录启动时激活，其他项目里完全静默。
  </Tab>
</Tabs>

#### 步骤 2c —— 让 Claude Code 验证两个东西都装好了

<PromptStep n={3} audience="Claude Code">

3. 跑捆绑的环境检查：`bash ~/.claude/skills/typst/scripts/verify-typst.sh`（按我挑的安装方式调整路径）。报告每行 `[PASS]` 是否都出现，CJK 字体是否被识别到。
</PromptStep>

<VerifyStep n={4}>

### 阶段 3 —— 从 profile 生成 CV

<PromptStep n={5} audience="Claude Code">
我想从 Neon 的 profile 行生成一页 CV。我们在阶段 2 装的 Typst skill 给你提供了模板和约定 —— 在动手写 `.typ` 之前，先去查它 `references/typst-templates.md` 里关于 resume / CV 的部分。然后请：

1. 读我的 `profile` 行（用 `.env.local` 里的 `DATABASE_URL`）。
2. 用 Typst skill 写一个干净的单列 CV 模板，放在 `cv/template.typ`：header 里是姓名 + headline、一段 summary、然后是 Education、Experience、Projects、Skills 区。用得体的 serif 正文 + sans 标题。强调色用我第 2 周挑的那个。
3. 在 `cv/build.ts` 写一个小 Node 脚本，读 DB 行、填 Typst 模板，然后 shell out 跑 `typst compile cv/template.typ cv/my-cv.pdf`。
4. 跑一次脚本。打开结果 PDF，把它的样子贴一段描述 / 截图给我。有任何不对劲的地方就迭代——但迭代通过 skill 做，永远不要手改 `.typ`。
</PromptStep>

<VerifyStep n={6}>

- `cv/my-cv.pdf` 存在、一页、里面是你真实 profile 数据。
- 标题、间距、字体都专业。
- 强调色和你的站点一致。

</VerifyStep>

### 阶段 4 —— 线上站点的 "Download CV" 按钮

<PromptStep n={7} audience="Cursor">
现在把它接到站里：

1. 在 `/api/cv` 加一个 serverless function，按需重新生成我的 CV PDF —— 逻辑和 Node 脚本一致，但跑在 Vercel function 里。流式返回 PDF 响应，`Content-Type: application/pdf`、`Content-Disposition: attachment; filename="ChanMeng-CV.pdf"`（把我的真实名字从 profile 行里替换进去）。
2. 在首页 hero 区现有社交链接旁边加一个 **Download CV** 按钮。点击触发下载。
3. 这个路由按 IP 限速每小时 20 次 —— CV 生成便宜但别浪费算力。复用第 7 周联系表单那套 rate-limit 模式。
4. 确保 Vercel function runtime 里有 `typst`。默认是没有的 —— 用 Typst skill 推荐的做法，可能是在 bundle 里带一个预编译二进制，或者走一个 fly.io / Railway 侧服务。
</PromptStep>

<VerifyStep n={8}>

- localhost：点 **Download CV** —— PDF 下载下来，带你的真实数据。
- 部署后生产：同一个按钮，同一个结果。
- 连点 21 次：第 21 次返回 429。

</VerifyStep>

### 阶段 5 —— 求职信生成

<PromptStep n={9} audience="Claude Code">
现在做求职信生成器。我要在站上有一个表单：粘进 JD、可选填公司名、点 **Generate cover letter**。它产出一份一页的定制 PDF，用我的声音（以 `content/persona.md` 为 voice guide）写，引用我 profile 里 experience 和 projects 数组里的 **真实** 条目。

具体步骤：

1. 在 `cv/cover-letter.typ` 写一个 Typst 模板——一页信布局：发件人 block（我）、收件人 block（公司名 + 未知则 "Dear Hiring Manager"）、正文段落、落款。字体语言与 CV 一致。
2. 写一个 server action：
   1. 接受 JD + 公司名 +（可选）具体岗位。
   2. 调用 Claude API，system prompt 写成：**"You write a crisp one-page cover letter in the voice of \[persona.md\]. Use only facts from the profile JSON provided — don't fabricate experience. Structure: opening hook, why this role, two specific project / experience references relevant to the JD, sign-off."** 返回结构化 JSON：`{ opening, body1, body2, body3, signoff }`。
   3. 把这个 JSON 填进 Typst 模板。
   4. 用 `typst compile` 编译，流式返回 PDF。
3. 在 `/edit-profile`（仅登录可见）加一个 **Generate cover letter** 区块：两个输入（公司、JD），一个 textarea，一个 **Generate** 按钮。成功后 PDF 下载。
4. 每次生成都记到一张新表 `cover_letters`（公司、岗位、created_at、正文长度）。我想以后回看自己申请过哪些。

在 `.env.local` 和 Vercel prod 里都设 `ANTHROPIC_API_KEY`。
</PromptStep>

<ManualStep n={10} why="Anthropic 的 console 要求人登录 + 接受条款后才能生成 key —— 每家主流 LLM provider 都一样。">
打开 [console.anthropic.com/settings/keys](https://console.anthropic.com/settings/keys)，登录，点 **Create Key**，命名 `technest-cover-letter`，复制，粘回 Claude Code / Cursor。
</ManualStep>

<VerifyStep n={11}>

- 粘一段真实 JD（比如 LinkedIn 上一条你可能真会投的）。
- 点 **Generate cover letter**。~20 秒后下载一页 PDF。
- 信里引用了你 profile 的真实内容（没有编）。
- 声音和 `persona.md` 一致。
- `cover_letters` 表里出现一行。

</VerifyStep>

<RecoverStep n={12}>
求职信生成了，但排版挤得慌 —— 正文溢到第二页。你能不能配合 Typst skill 微调模板，让内容总在一页内，即使 JD 较长？优先压缩段落，而不是把字号调到 10 pt 以下。
</RecoverStep>

## 真实世界应用

招聘时间线还会继续被压缩。那个能在看到岗位后 1 小时内，发出一份**具体、有思考、排版清爽**的求职信的候选人，每一次都会赢过那个花两天写的人。你现在手里就是这台机器。留住它、打磨它，以后再也别给"CV builder SaaS"付钱。

<Note title="Career">

## 踩坑提醒 & 小贴士

- *"Vercel function runtime 里没有 Typst。"* 两条路——(a) 把预编译二进制打进 function bundle（~5 MB），(b) 把编译外置到一个 tiny Fly.io / Railway 侧服务。Typst skill 知道这两种模式——让它按你 Vercel 套餐挑合适的那一种。
- *"求职信臆造了我没待过的公司。"* 加强 system prompt：**"Cite only items explicitly present in `experience` or `projects` arrays. If there's nothing relevant, say 'While I haven't worked at X directly, …' and pivot to the closest real item."**
- *"PDF 字体怪怪的。"* Vercel runtime 可能没有 Inter / 你选的字体。Typst skill 会把字体嵌进 bundle —— 问它：**"Ensure the CV uses an embedded font so it renders identically on my machine and on Vercel."**
- *"我求职信在 Vercel 上强调色不对。"* 你在两个地方硬编码了主题值。问：**"Factor the theme into one Typst variable and reference it from both CV and cover-letter templates."**
- *"10 次测试后 Claude 太贵了。"* 让 Cursor 对相同的 (JD, 公司) 组合做 24 小时缓存——你测试时大多是同样的输入。

<Info title="如果 Typst 在生产环境编译挂了、报错又看不懂">
在本地用完全相同的输入再跑一遍 —— 95% 的情况 Typst 本地会出同样的错，你能让 Claude Code（带 Typst skill）修。如果只在 Vercel 挂，那是 runtime / 字体 / 二进制问题，skill 知道怎么诊断。
</Info>
