# 图表在不同场景的应用指南

## 学术论文图表规范

### 期刊投稿要求

<Admonition type="important" title="通用学术规范">
- **矢量格式**：使用 SVG 或 EPS 格式，确保打印质量
- **字体规范**：Times New Roman 或 Arial，大小不小于 8pt
- **编号规则**：Figure 1, Figure 2... 连续编号
- **引用格式**：在正文中使用 "如图 1 所示" 或 "见图 1"
- **分辨率要求**：位图至少 300 DPI
- **颜色模式**：打印用 CMYK，电子版用 RGB
</Admonition>

### IEEE 论文图表规范

**尺寸要求**：

- 单栏：3.5 英寸宽
- 双栏：7.16 英寸宽
- 高度：不超过 9.5 英寸

**标注规范**：

- 图题：图下方，居中
- 字体：8-10pt
- 线宽：0.5-1pt

### API 文档图表

**推荐图表类型**：

- **时序图**：展示 API 调用流程，适合认证流程、支付流程
- **流程图**：说明业务逻辑，适合数据处理、决策流程
- **状态图**：描述资源状态变化，适合订单状态、任务状态

### 系统设计文档

<Admonition type="tip" title="图表组织建议">
1. **分层展示**：从高层架构到详细设计逐层深入
2. **统一风格**：保持配色、字体、图标风格一致
3. **版本管理**：图表源文件纳入 Git 管理
4. **更新同步**：代码变更时及时更新图表
</Admonition>

## PPT 演示优化

### 分步展示技巧

**渐进式展示**：

1. 第 1 页：核心概念
2. 第 2 页：添加细节
3. 第 3 页：完整架构
4. 第 4 页：关键点强调

**动画效果**：

- 淡入：新增组件
- 路径动画：数据流向
- 颜色变化：状态改变
- 缩放：焦点切换

## 响应式设计

### 移动端优化

<Admonition type="tip" title="移动端图表设计原则">
- **简化内容**：减少节点数量，突出核心信息
- **增大元素**：确保触摸目标至少 44×44 像素
- **垂直布局**：优先使用从上到下的布局
- **可缩放**：使用 SVG 格式支持缩放
- **分段展示**：复杂图表拆分为多个简单图表
</Admonition>
