# Mermaid 基础教程

## 什么是 Mermaid

Mermaid 是一个基于文本的图表生成工具，通过简单的文本描述就能生成各种专业图表。它的核心理念是"代码即图表"，让图表的创建和维护变得像编写代码一样简单。

### 核心优势

- **版本控制友好**：纯文本格式，可以使用 Git 管理
- **易于维护**：修改图表只需编辑文本
- **平台通用**：支持 GitHub、GitLab、Notion、Obsidian 等多个平台
- **实时预览**：编写即可见，所见即所得
- **无需设计技能**：专注于内容，样式自动处理

## 流程图（Flowchart）

流程图是最常用的图表类型，用于展示过程、算法或工作流程。

### 基础语法

#### 图表方向

<CodeBlock language="mermaid">
{`graph TD  %% Top Down - 从上到下
graph LR  %% Left to Right - 从左到右
graph BT  %% Bottom to Top - 从下到上
graph RL  %% Right to Left - 从右到左`}
</CodeBlock>

#### 节点形状

<CodeBlock language="mermaid">
{`graph LR
    A[矩形节点]
    B(圆角矩形)
    C([体育场形])
    D[[子程序形]]
    E[(数据库形)]
    F((圆形))
    G{菱形判断}
    H{{六边形}}
    I[/平行四边形/]
    J[\\反向平行四边形\\]`}
</CodeBlock>

```mermaid
graph LR
    A[矩形节点]
    B(圆角矩形)
    C([体育场形])
    D[[子程序形]]
    E[(数据库形)]
    F((圆形))
    G{菱形判断}
    H{{六边形}}
    I[/平行四边形/]
    J[\反向平行四边形\]
```

#### 连接线样式

<CodeBlock language="mermaid">
{`graph LR
    A --> B     %% 实线箭头
    C --- D     %% 实线
    E -.-> F    %% 虚线箭头
    G -.- H     %% 虚线
    I ==> J     %% 粗线箭头
    K === L     %% 粗线`}
</CodeBlock>

```mermaid

    A --> B
    C --- D
    E -.-> F
    G -.- H
    I ==> J
    K === L
```

## 时序图（Sequence Diagram）

时序图用于展示对象之间的交互顺序，特别适合描述系统组件间的通信过程。

### 基础语法

<CodeBlock language="mermaid">
{`sequenceDiagram
    participant A as Alice
    participant B as Bob

    A->>B: 实线箭头消息
    B-->>A: 虚线箭头响应
    A->>+B: 激活 B
    B->>-A: 停用 B

    Note over A: 单个参与者注释
    Note over A,B: 跨参与者注释`}
</CodeBlock>

```mermaid
sequenceDiagram
    participant A as Alice
    participant B as Bob

    A->>B: 实线箭头消息
    B-->>A: 虚线箭头响应
    A->>+B: 激活 B
    B->>-A: 停用 B

    Note over A: 单个参与者注释
    Note over A,B: 跨参与者注释
```

### 高级特性

#### 循环和条件

<CodeBlock language="mermaid">
{`sequenceDiagram
    participant U as 用户
    participant S as 服务器
    participant D as 数据库

    U->>S: 登录请求

    alt 凭证正确
        S->>D: 查询用户信息
        D-->>S: 返回用户数据
        S-->>U: 登录成功
    else 凭证错误
        S-->>U: 登录失败
    end

    loop 每5秒
        U->>S: 心跳检测
        S-->>U: 在线状态
    end`}
</CodeBlock>

## 最佳实践

**命名规范**：

- 使用有意义的节点ID
- 保持标签简洁明了
- 避免过长的文本

**布局优化**：

- 选择合适的图表方向
- 控制图表复杂度
- 合理使用子图分组

**视觉设计**：

- 保持样式一致性
- 使用颜色区分类别
- 避免过度装饰
