## Project 2: PATH Environment Optimization

Messy PATH configuration can lead to command conflicts and performance issues. Let's use AI to optimize it.

### Common PATH Problems

- **Duplicate Paths**: The same path is added multiple times, affecting lookup efficiency
- **Invalid Paths**: Pointing to non-existent directories, causing command lookup failures
- **Order Confusion**: Improper priority settings may use wrong tool versions

### Optimization Practice

<Tabs>
<TabItem value="analyze" label="Analyze PATH" default>

<CodeBlock language="text">
{`You: Help me analyze the system PATH and find all problems

Gemini: Analyzing your PATH configuration...

🔍 PATH Analysis Report:

Current PATH entries (in order):
1. /usr/local/bin ⚠️
2. /opt/homebrew/bin ✅
3. /usr/local/bin ⚠️ [Duplicate]
4. ~/.npm-global/bin ✅
5. /usr/bin ✅
6. /opt/old-tools/bin ❌ [Does not exist]
7. ~/.npm-global/bin ⚠️ [Duplicate]
8. /usr/local/go/bin ✅
9. /Applications/Docker.app/Contents/Resources/bin ❌ [Docker not installed]
10. /usr/local/bin ⚠️ [3rd duplicate]

📊 Problem Summary:
- 3 duplicate entries
- 2 invalid paths
- Recommended to adjust priority order

🎯 Optimization Recommendations:
1. User-level tools should take priority over system-level
2. Commonly used tool paths should be at the front
3. Remove all duplicate and invalid entries`}
</CodeBlock>

</TabItem>
<TabItem value="optimize" label="Execute Optimization">

<CodeBlock language="text">
{`You: Please help me optimize PATH configuration

Gemini: I'll help you optimize PATH configuration.

📝 Optimization Plan:

New PATH order (from highest to lowest priority):
1. ~/.local/bin          # User custom scripts
2. /opt/homebrew/bin     # Homebrew (macOS)
3. ~/.npm-global/bin     # npm global packages
4. /usr/local/go/bin     # Go language
5. /usr/local/bin        # Local installations
6. /usr/bin              # System commands

🔧 Applying changes...

Detected you're using zsh, updating ~/.zshrc:

✓ Backed up original .zshrc to .zshrc.backup
✓ Cleaned up duplicate export PATH statements
✓ Added optimized PATH configuration
✓ Added comments explaining each path's purpose

export PATH="/opt/homebrew/bin:$PATH"
export PATH="$HOME/.npm-global/bin:$PATH"
export PATH="/usr/local/go/bin:$PATH"

🎉 Optimization complete!

Run the following command to apply changes:
source ~/.zshrc

Or reopen your terminal window.`}

Problem: Module path alias configuration error
Cause: TypeScript and Webpack path aliases are inconsistent

Solution:
1. Update tsconfig.json:
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    }
  }
