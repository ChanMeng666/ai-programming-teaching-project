# 部署到Vercel

## 部署准备

<div className="deployment-prep">

### 检查清单
1. 确保代码已推送到GitHub
2. 检查`docusaurus.config.js`中的配置
3. 确认所有依赖都已正确安装

:::tip 部署前检查
- 本地运行构建命令确保无错误
- 检查环境变量配置
- 确认网站基本信息配置
:::

</div>

## Vercel部署流程

<div className="deployment-steps">

### 部署步骤

<Tabs>
  <TabItem value="setup" label="初始设置" default>
    1. 访问 https://vercel.com/
    2. 使用GitHub账号登录
    3. 点击"New Project"
    4. 从GitHub导入项目
  </TabItem>
  <TabItem value="config" label="项目配置">
    1. 选择你的博客仓库
    2. 点击"Import"
    3. 配置项目：
       - Framework Preset: 选择"Other"
       - Build Command: `npm run build`
       - Output Directory: `build`
    4. 点击"Deploy"
  </TabItem>
</Tabs>

</div>

## 自定义域名配置

<div className="domain-setup">

### 添加域名

1. 在Vercel项目页面中：
   - 点击"Settings"
   - 选择"Domains"
   - 点击"Add Domain"

### DNS配置
```bash
# A记录配置
Type: A
Name: @
Value: 76.76.21.21

# CNAME配置
Type: CNAME
Name: www
Value: cname.vercel-dns.com
```

:::caution 注意事项
- DNS生效可能需要几分钟到几小时
- 确保域名已经完成实名认证
- 检查DNS记录是否正确配置
:::

</div>
